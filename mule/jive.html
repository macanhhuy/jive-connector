Configuration

To use the this module within a flow the namespace to the module must be included. The resulting flow will look similar to the following:

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xmlns:jive="http://www.mulesoft.org/schema/mule/jive"
      xsi:schemaLocation="
               http://www.mulesoft.org/schema/mule/core
               http://www.mulesoft.org/schema/mule/core/current/mule.xsd
               http://www.mulesoft.org/schema/mule/jive
               http://www.mulesoft.org/schema/mule/jive/current/mule-jive.xsd">

      <!-- here goes your flows and configuration elements -->

</mule>

This module is configured using the config element. This element must be placed outside of your flows and at the root of your Mule application. You can create as many configurations as you deem necessary as long as each carries its own name.

Each message processor, message source or transformer carries a config-ref attribute that allows the invoker to specify which configuration to use.

<jive:create>

Creates an entity of the given type. If the entity already exists or cannot be created the system throws a JiveGenericException with the correspondat message. If the data pass in entity it's not the data expected the system will throw a JiveGenericException with the correspondant message.

XML Sample
<jive:create type="BLOG">
	<jive:entity>
     	<creationDate>#[variable:creationDate]</creationDate>
        <author>#[variable:author]</author>
    </jive:entity>
</jive:create>

Attributes
NameDefault ValueDescription
config-ref Optional. Specify which configuration to use.
type The type of entity to create
Child Elements
<jive:entity>
The entity attributes
Return Payload
  • The jive response as a Map. Generally it will return the entity just created with additional data added by jive.

<jive:delete>

Deletes an existent entity If there is no entity with the given id the system will throw a JiveGenericException with the message returned from the server. If the number of arguments is wrong the system will throw a JiveGenericException

XML Sample
<jive:delete type="BLOG" id="#[map-payload:blogId]" />

Attributes
NameDefault ValueDescription
config-ref Optional. Specify which configuration to use.
type The entity type
id The id of the entity to delete
Child Elements

<jive:execute>

Executes a custom operation

XML Sample
<jive:execute operation="AVATAR_GET_GLOBAL_AVATARS"/>
If a wrong number of attributes is sent, or the operation requires an entity an none is sent, the system will repond with a JiveGenericException.

Attributes
NameDefault ValueDescription
config-ref Optional. Specify which configuration to use.
operation The custom operation to be executed
id Optional. The path parameters to be added to the operation uri
Child Elements
<jive:entity>
Optional. The Map with the entity data to be send as payload
Return Payload
  • A Map with the server response

<jive:get>

Retrieves the attributes for the entity of the given type and id.

XML Sample
<jive:get type="AVATAR" id="#[map-payload:avatarId]"/>
If the entity you are trying to get doesn't exist, the system will throw a JiveGenericException with the correponging message.
